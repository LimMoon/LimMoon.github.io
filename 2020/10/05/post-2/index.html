Spring

Inversion of Control 과 Dependency Injection 을 대표적으로 지원하는 java opensource framework for application development.

  • IoC (Inversion of Control): framework가 코드를 불러주는 디자인. 개발자가 짠 custom code가 라이브러리를 부르던 traditional 패턴과 반대되는 개념.
  • DI (Dependency Injection): 클라이언트가 디펜던시를 부르는 대신 injector가 클라이언트에게 어떤 서비스를 쓸 지 결정해준다. IoC 패턴에서 쓰이는 테크.
  • Spring Boot: autoconfiguration 을 지원하여 쉽게 spring application을 개발하게 도와주는 module.
  • Spring MVC: Spring 의 web MVC framework. MVC 패턴은, 사용자의 조작을 controller로 받아오고, controller는 model을 통해서 데이터를 가져오고 view는 그 정보를 바탕으로 시각적으로 표현하여 사용자에게 전달한다.

REST api

  • RESTful 이란: representational state transfer (REST) is an architectural style for an application program interface (API) that uses HTTP requests to access and use data. That data can be used to GET, PUT, POST and DELETE data types, which refers to the reading, updating, creating and deleting of operations concerning resources.
  • SOAP방식과 REST방식의 차이점은?
    SOAP(Simple Object Access Protocol) HTTP, HTTPS, SMTP 등을 통해 XML 기반의 메시지를 컴퓨터 네트워크 상에서 교환하는 프로토콜이다. rigid protocol for different programs to be able to communicate. only work with xml formats.
    REST(Representational State Transfer) was designed for working with media components, files etc. architectural pattern. flexible, works wih plain text, xml, html, json

Java

  • String buffer vs string builder
    stringbuffer is threadsafe, but less efficient.
  • Java vs C
    java 는 jvm 을 사용하여 os independent. compiled byte code (.class file) is platform independent.
    c is procedural programming language, following top down approach. program is divided into smaller functions. java is object oriented. program is divided into classes, and classes provide data hiding.
  • garbage collection
    java has automatic memory management. deletes unreferenced data on the heap
  • hashCode method? equals 차이
    equals() method is used to compare equality of two Objects
    hashCode() method returns the hashcode value as an Integer. Hashcode value is mostly used in hashing based collections like HashMap, HashSet, HashTable.. This method must be overridden in every class which overrides equals() method.
  • jpa vs hibernate
    jpa (Java persistence API) is like a bridge between java model class and relation db layer. jpa is a standard way of ORM (Object relational mapping). Hibernate is an implementation of jpa, an ORM tool. this framework is used to store java objects in relational db system.
  • JDBC vs ODBC
    ODBC: open database connectivity, for window platform
    JDBC: JAVA database connectivity, for java language. JAVA API to connect and execute the query with database

    stack memory allocation happens during function call stack. size of allocation is known to compiler. when function call is over, stack gets popped. heap memory is allocated during execution of instruction written by the programmer. need to be allocated and deallocated, otherwise memory leak can happen

OOP

  • object oriented vs procedure oriented
    c is procedural programming language, following top down approach. program is divided into smaller functions. java is object oriented. program is divided into classes, and classes provide data hiding.
  • inheritance
    Inheritance is one in which a new class is created that inherits the properties of the already exist class. It supports the concept of code reusability and reduces the length of the code in object-oriented programming.
  • polymorphism
    Polymorphism is that in which we can perform a task in multiple forms or ways. It is applied to the functions or methods. Polymorphism allows the object to decide which form of the function to implement at compile-time as well as run-time.
    Types of Polymorphism are:
    • Compile-time polymorphism (Method overloading)
    • Run-time polymorphism (Method Overriding)
  • override vs overload
    Overriding means having two methods with the same method name and parameters (i.e., method signature). One of the methods is in the parent class and the other is in the child class. Overriding allows a child class to provide a specific implementation of a method that is already provided its parent class. Overloading occurs when two or more methods in one class have the same method name but different parameters.
  • interface vs abstract
    An abstract class allows you to create functionality that subclasses can implement or override. An interface only allows you to define functionality, not implement it. And whereas a class can extend only one abstract class, it can take advantage of multiple interfaces. A Java interface can be implemented using keyword “implements” and abstract class can be extended using keyword “extends”. Members of a Java interface are public by default. A Java abstract class can have class members like private, protected, etc.n interface can extend another Java interface only, an abstract class can extend another Java class and implement multiple Java interfaces.
